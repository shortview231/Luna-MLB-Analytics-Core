# Luna_Export

Luna_Export is the staging repository for export-ready artifacts generated by Luna workflows.

This repository exists to support a controlled pipeline:

`Generate -> Stage -> Approve -> Export`

Phase 1 is intentionally limited to export staging infrastructure only. It does not include watcher automation, Gmail integration, auto-commit flows, GitHub Pages automation, or multi-artifact bundles.

## Lifecycle

- `exports/pending/`: generated artifacts waiting for review and approval
- `exports/approved/`: artifacts cleared for downstream publishing or export
- `exports/archived/`: artifacts retained for history after their active use ends

Approval is required before any artifact should be treated as public-safe or ready for downstream push/export.

## Supported Artifact Classes

Phase 1 scaffolds the repository for:

- LinkedIn posts
- GitHub exports
- screenshots
- daily reports
- portfolio artifacts
- email artifacts

## Metadata

Artifact metadata is stored separately from artifact files:

- Schema: `metadata/schemas/export_artifact.schema.json`
- Records by lifecycle state:
  - `metadata/records/pending/`
  - `metadata/records/approved/`
  - `metadata/records/archived/`

Each artifact should have one sidecar metadata record describing status, approval state, and related files.

## Phase 1 Approval Flow

The initial approval flow is manual-triggered but script-backed for consistency.

Approve an artifact by file path:

```bash
python3 scripts/approve_artifact.py exports/pending/linkedin/2026-04-01-linkedin-post.md
```

Approve an artifact by metadata id:

```bash
python3 scripts/approve_artifact.py linkedin-2026-04-01-devlog
```

The script will:

1. find the pending metadata record
2. verify the artifact is still pending
3. move the artifact into the mirrored `exports/approved/` path
4. move the metadata record into `metadata/records/approved/`
5. update metadata fields so `status` becomes `approved` and `approved` becomes `true`
